# 103 — Prestart Temperature Alert

## Description
Alarm code **103** is generated by the Carrier Pro-Dialog Plus controller when the **leaving chilled water temperature** or **ambient temperature** is outside the permissible startup window during the pre-start self-check sequence. Unlike E3 and E5, this is typically an **auto-reset** alert that delays unit startup until conditions normalize.

## Common Conditions

| Scenario | Cause | Action |
|----------|-------|--------|
| Cold ambient startup (<40°F) | Low ambient lockout enabled | Wait for ambient to rise, verify low-ambient kit installed |
| Leaving water temp too high | Warm loop startup | Normal — unit will start when temperature is detected |
| Leaving water temp too low | Freeze protection active | Check chilled water setpoint vs. freeze limit |
| Oil migration | Crankcase heater failed | Replace heater, allow 8-hour pre-heat |

## Step-by-Step Repair Guide
1. Check controller display for specific sub-alarm code within fault 103
2. Verify ambient temperature sensor reading vs. actual ambient
3. Check leaving water temperature sensor accuracy
4. Inspect crankcase heater operation — should be warm to touch when unit is off
   - Measure heater continuity and voltage supply
5. If sensor drift suspected → verify with [[sops/sop-electrical-safety]] and calibrated reference
6. Check Pro-Dialog setpoints: Low Ambient Cutout, Freeze Protection Limit, Startup Delta T
7. Document and clear alarm after conditions normalize
